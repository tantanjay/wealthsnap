# WealthSnap v1.1.1 - Release Notes

**Release Date:** January 21, 2026
**Build:** Version 1.1.1 (Build 4)

---

## ✨ What's New

### 🎨 UX Improvements
- **Transaction Options Redesign**
  - Replaced the native alert menu with a modern **Bottom Modal** for transaction options (Edit/Delete) in the History screen.
  - Improved accessibility and visual consistency with the rest of the app.
- **Enhanced Tap Targets**
  - Fixed an issue where transaction cards in the History list had small or inconsistent tap areas.
  - Interaction is now smoother and more reliable across the entire card width.
- **Improved Record Menu Workflow**
  - **Re-clickable Record Menu**: The Record menu/tab can now be tapped again while in the Transaction Form to reopen the selection menu.
  - **Auto-Reset Form**: Switching transaction types (e.g., Expense to Income) now completely resets the form to a fresh state.
  - **Android Back Support**: Pressing the hardware back button while in the form now returns to the menu instead of exiting.
- **Visual Polish**
  - Added border styling to the Transaction Options preview card for better separation.
- **Improved Modal Experience**
  - **Dynamic Calculator Sizing**: The calculator modal now adapts to screen size (taller on small screens) and supports scrolling.
  - **Recurring Transactions Modal**:
    - **Vertical Layout**: Active switch and delete button are now stacked vertically for better handling of long rule names.
  - Replaced system-native alerts with a themed custom modal for a unified design.
  - Applied to "Coming Soon", "Delete Confirmation", "Success", and "Error" messages across the app.

### 🤖 AI Integration
- **Smart Receipt Analysis**
  - **Auto-Transaction**: Upload or capture a receipt, and AI will automatically analyze it to create a transaction.
  - **Smart Categorization**: Automatically determines the category based on spending breakdown.
  - **Contextual Notes**: Generates a detailed note including Merchant name and item summary.
- **Enhanced AI UI**
  - **Secure Loading State**: Added a "Processing..." overlay that locks the UI (no accidental dismissal) during analysis.
  - **Visual Indicator**: Transactions created by AI are marked with a ✨ (sparkles) icon in the History list.

### 🐛 Bug Fixes
- **Duplicate Recurring Transactions**
  - Fixed a critical bug where editing a recurring transaction would erroneously create a duplicate recurrence rule.
  - Editing a recurring entry now correctly updates the existing rule without duplication.
- **PIN Setup Double Alert**
  - Fixed an issue where the "Success" alert would appear twice when creating or changing a PIN.
  - Resolved inconsistent callback handling during the onboarding flow.

### 🔐 Security & Onboarding
- **Unified PIN Creation UI**
  - Standardized the PIN entry screen design to look consistent across both Onboarding and Profile Settings.
  - Fixed layout issues where the keyboard could appear compressed on smaller screens.

---

## 🛠️ Technical Changes
- **Refactoring**: Moved existing transaction option logic to a reusable `TransactionOptionsModal` component.
- **Alert System**: Implemented a global `AlertContext` and `CustomAlert` component to replace React Native's `Alert.alert`.
- **Code Quality**: Resolved linting warnings related to component props and styling consistency.
- **Stability**: Fixed a circular dependency between `storageService` and `dataCache` to improve application stability and build warnings.
- **AI Backend**: Refactored AI services to separate configuration and logging logic, preparing for multi-provider support.

2. **Editing**: Edit a recurring transaction and verify that no duplicate entries appear in the future.

# WealthSnap v1.5.1 - Release Notes

**Release Date:** January 27, 2026  
**Build:** Version 1.5.1 (Performance & Data Security Update)

> [!TIP]
> **🚀 PERFORMANCE BREAKTHROUGH**
> 
> This update specifically targets performance bottlenecks for users with large transaction histories. We've completely re-engineered how the app "breathes" during data loading.
>
> **Key Impacts:**
> 1. **Smoother App Launch**: The "Home Screen Freeze" has been eliminated.
> 2. **Background Security**: Decryption now happens in the background, keeping the UI interactive.
> 3. **Lower-End Device Support**: Significant performance gains on mid-to-low range hardware.

---

## ✨ What's New

### ⚡ Non-Blocking "Marathon" Decryption
We've introduced a major architecture update to handle massive ledgers (10,000+ transactions) without locking up the user interface.

#### 🧠 Smart Chunking
- **Background Processing**: Instead of one massive "freeze," the app now decrypts data in controlled batches of 500 records.
- **Thread Yielding**: Between batches, the app gives control back to the UI thread for a split second.
- **Interactive Loading**: You can now tap the **"Add Transaction"** button or navigate the menu **while** your full history is still decrypting in the background.

#### 🦴 Professional Skeleton States
- **Visual Feedback**: Replaced static "frozen" screens with high-performance Skeleton Shimmers.
- **Status Awareness**: See exactly where the app is "thinking" without losing the ability to interact with the rest of the UI.

---

## 🛠️ Technical Changes

### 🔒 High-Performance Encryption Service
- **Refactored `bulkDecryptItems`**: Moved bulk decryption logic into a centralized, non-blocking utility within the `EncryptionService`.
- **Async Event Loop Management**: Implemented `setTimeout(0)` yielding logic to prevent JavaScript thread starvation.


---

## 🐛 Bug Fixes
- **UI Lockup (ANR)**: Fixed a critical issue where older devices would trigger an "App Not Responding" error when loading more than 5,000 transactions.

---

## 📝 For Testers

Please focus your testing on **Performance & Responsiveness**:

1. **The "Start-up Marathon"**:
   - Open the app with a large dataset.
   - Verify that you can **immediately** tap the "Add Transaction" button while the balance cards are still showing skeleton shimmers.
   
2. **Low-End Device Stability**:
   - Test on devices with 4GB-6GB RAM.
   - Verify the app doesn't show a "System Unresponsive" dialog during the initial 8-10 second load.

3. **Optimistic Deletion**:
   - Delete a transaction and verify it disappears **instantly** from the list, even before the database confirms the deletion.
